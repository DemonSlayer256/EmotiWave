# Data Pipeline and Design Decisions

## Overview

This project performs EEG-based emotion recognition using the DREAMER dataset. The objective is to classify emotional states from EEG recordings using Differential Entropy (DE) features and an XGBoost classifier.

The data pipeline is intentionally divided into multiple stages to ensure reproducibility, maintainability, and clear separation of concerns.

---

# Data Directory Structure

```text
data/
├── raw/
│   ├── DREAMER.mat
│   └── metadata.json
│
├── interim/
│   ├── metadata.json
│   ├── S01_T01.npy
│   ├── S01_T02.npy
│   └── ...
│
├── processed/
│   └── .gitkeep
```

## Rationale

The directory structure follows a standard machine learning workflow:

* **raw/** contains original downloaded data and dataset documentation.
* **interim/** contains extracted and structured EEG trials.
* **processed/** contains machine-learning-ready feature datasets generated after feature extraction.

This separation allows every stage of the pipeline to be reproduced independently.

# Data Extraction

## process_data.py

Purpose:

Convert DREAMER.mat into a structured format suitable for downstream processing.

The script:

1. Loads DREAMER.mat.
2. Extracts EEG stimulus recordings.
3. Extracts labels.
4. Extracts subject metadata.
5. Saves individual EEG trials as NumPy files.
6. Creates a metadata index.

Output:

```text
data/interim/
├── metadata.json
├── S01_T01.npy
├── S01_T02.npy
└── ...
```

---

# Why EEG Trials Are Saved Separately

Instead of storing the entire dataset in one file, each trial is saved independently.

Example:

```text
S01_T01.npy
```

Meaning:

```text
Subject 01
Trial 01
```

Advantages:

* Faster loading during feature extraction.
* Easier debugging.
* Reduced memory usage.
* Individual trial inspection.
* Simpler experimentation.

```json
    "age": 22,
    "gender": "male",
    "valence": 4.0,
    "arousal": 3.0,
    "dominance": 2.0,
    "valence_class": 1,
    "arousal_class": 0,
    "dominance_class": 0
}
```

Advantages:

* Links EEG files to labels.
* Avoids repeatedly parsing DREAMER.mat.
* Provides a single source for metadata lookup.

# Label Generation Strategy

The DREAMER dataset provides ratings on a 1–5 scale.

## Initial Approach

```python
label = int(score >= 3)
```

This treated the midpoint rating (3) as a positive class.

Observed distributions:

```text
Valence   : {1: 253, 0: 161}
Arousal   : {1: 300, 0: 114}
Dominance : {1: 319, 0: 95}
```

Significant class imbalance was observed.

---

## Final Decision

```python
label = int(score > 3)
```

This treats:

```text
1, 2, 3 → Low
4, 5    → High
```

Rationale:

* Rating 3 represents a neutral midpoint.
* Produces substantially more balanced classes.
* Reduces classifier bias toward the majority class.
* Better reflects the psychological interpretation of ratings.

Observed distributions:

```text
Valence   : {0: 251, 1: 163}
Arousal   : {0: 233, 1: 181}
Dominance : {0: 215, 1: 199}
```

This strategy was adopted for all subsequent experiments.

# Why This Data Is Stored in interim/

The extracted EEG files are not yet machine-learning-ready.

No:

* Baseline correction
* Band-pass filtering
* Artifact removal
* Feature extraction

has been performed.

Therefore these files represent:

```text
Structured Data
```

rather than:

```text
Processed Data
```

The `interim/` directory accurately reflects this stage of the pipeline.
